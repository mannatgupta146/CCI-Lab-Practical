Practical 1: Network Traffic Analysis (Wireshark)

How-To: Network Traffic Analysis

  • Purpose: To capture and analyze live network traffic ‚Äî see how data packets move between your system and the internet.
  • Software Used: Wireshark, Web Browser
  • How to Perform:
    1. Open Wireshark ‚Üí select your active network interface (Wi-Fi/Ethernet).
    2. Click Start Capture.
    3. Visit a few websites (e.g., google.com, example.com).
    4. Stop capture and use filters like: http, dns, tcp.
    5. Observe source/destination IP, port numbers, and protocols.
  • Output: List of captured packets showing protocols (HTTP, TCP, DNS), IP addresses, and detailed info per packet.

  • Packet, Protocol (TCP, UDP, HTTP, DNS)
  • TCP 3-Way Handshake (SYN, SYN-ACK, ACK)
  • DNS Query, HTTP GET/POST
  • IP Address (Source/Destination), Port Number (80, 443)
  • üß∞ Software: Wireshark (Packet capturing and analysis), Web Browser (To generate traffic).

Practical 3: Deleted File Recovery (Autopsy)

How-To: Extraction & Analysis of Deleted Files

  • Purpose: To recover permanently deleted files and understand how forensic tools analyze disk images.
  • Software Used: FTK Imager (for disk imaging), Autopsy (for analysis)
  • How to Perform:
    1. In FTK Imager ‚Üí File ‚Üí Create Disk Image ‚Üí select a USB or partition.
    2. Save as .E01 (forensic image).
    3. Open Autopsy ‚Üí New Case ‚Üí Add Data Source ‚Üí Disk Image.
    4. Enable modules like ‚ÄúDeleted File Finder‚Äù & ‚ÄúFile Type Identification.‚Äù
    5. Analyze recovered deleted files (red X mark).
    6. Right-click ‚Üí ‚ÄúExtract File(s)‚Äù to restore.
  • Output: List of deleted/recovered files with timestamps (MAC times) and content preview.

üß† Important Terms & Tools

  • Digital Evidence, Disk Image (E01)
  • Deleted File Recovery, File Carving
  • MAC Times (Modified, Accessed, Created)
  • Unallocated Space
  • üß∞ Software: FTK Imager (For creating images), Autopsy (For analysis).
Practical 4: Log Analysis (Windows & Linux)

How-To: Log Analysis (Windows & Linux)

  • Purpose: To detect unauthorized access attempts using system logs.
  • Software Used: Windows Event Viewer / PowerShell, Linux journalctl, grep, awk, auth.log
  • How to Perform (Windows):
    1. Run eventvwr.msc.
    2. Check Windows Logs ‚Üí Security.
    3. Filter by Event IDs: 4624 (Success), 4625 (Failure), 4740 (Lockout).
  • How to Perform (Linux):
    1. Open terminal.
    2. Run: sudo grep "Failed password" /var/log/auth.log
    3. Run: sudo last (shows last logins).
  • Output: Identified failed logins, login times, and users/IPs involved.

üîç Important Terms & Tools

  • Event Logs, Sysmon, Audit Policy
  • journalctl, /var/log/auth.log
  • Event IDs (Windows): 4624, 4625, 4740
  • Unauthorized Access Detection
  • üß∞ Software: Windows Event Viewer, PowerShell, journalctl, grep, auditd.
Practical 5: Mobile Forensics (MOBILedit)

How-To: Mobile Forensics

  • Purpose: To extract and analyze data (contacts, SMS, call logs, media) from mobile devices.
  • Software Used: MOBILedit Forensic Express, Autopsy
  • How to Perform:
    1. Connect mobile to PC ‚Üí enable USB Debugging (for Android).
    2. Open MOBILedit ‚Üí select Extract Phone Data.
    3. Choose type: Logical (messages, contacts) or Physical (entire dump).
    4. Export extracted data (CSV/XML).
    5. Open Autopsy ‚Üí import exported folder ‚Üí analyze artifacts.
  • Output: List of extracted communication artifacts (contacts, call history, messages).

üì± Important Terms & Tools

  • Mobile Forensics, Logical Acquisition, Physical Acquisition
  • Artifacts (SMS, calls, media, app data)
  • EXIF Data (from images)
  • üß∞ Software: MOBILedit Forensic Express, Autopsy, USB Debugging (Android).
Practical 6: Cyber Attack Analysis (Wireshark)

How-To: Cyber Attack Forensic Analysis

  • Purpose: To analyze a .pcap file and identify attack types like brute force, port scanning, or data theft.
  • Software Used: Wireshark
  • How to Perform:
    1. Open Wireshark ‚Üí File ‚Üí Open ‚Üí select .pcap file.
    2. Go to Statistics ‚Üí Protocol Hierarchy, Endpoints, Conversations.
    3. Use filters: tcp.flags.syn == 1 && tcp.flags.ack == 0 (Port scan), ftp, telnet.
    4. Identify attacker IP and activity pattern.
    5. Create timeline of attack.
  • Output: List of attacker IPs, protocols used, suspicious requests, and sequence of events.

üïµÔ∏è Important Terms & Tools

  • .pcap File, Port Scan, Brute Force Attack
  • Data Exfiltration, Command & Control (C2)
  • Xmas Scan / SYN Scan
  • üß∞ Software: Wireshark.
Practical 7: Email Threat Analysis (Phishing)

How-To: Email Threat Analysis

  • Purpose: To demonstrate how advanced email detection techniques identify phishing and malware.
  • Software Used: Kali Linux (SET toolkit), Python (smtplib), Thunderbird/Outlook, Email Security Solutions
  • How to Perform:
    1. Use SET or Python to send crafted phishing emails to test inbox.
    2. Include fake login links or attachments.
    3. Observe detection by the security filter.
    4. Check for SPF/DKIM/DMARC warnings in the email header.
    5. Note whether the email was delivered, quarantined, or blocked.
  • Output: Logs or screenshots showing detection of phishing/BEC/malware attempts.

Practical 8: Browser Privacy Analysis

How-To: Data Privacy Analysis

  • Purpose: To study how websites track users and how privacy tools prevent tracking.
  • Software Used: Google Chrome / Firefox / Brave, Ghostery, Privacy Badger, uBlock Origin
  • How to Perform:
    1. Open browser ‚Üí Developer Tools (F12) ‚Üí Network tab.
    2. Visit a tracking-heavy website (news, e-commerce).
    3. Note third-party requests and cookies created.
    4. Install privacy extensions ‚Üí revisit same website.
    5. Observe fewer requests and blocked trackers.
  • Output: Comparison showing number of trackers/cookies before and after enabling privacy tools.

üåê Important Terms & Tools

  • Cookies, Local Storage / Session Storage
  • Third-Party Trackers, Fingerprinting
  • Incognito Mode
  • üß∞ Software: Browsers (Chrome, Firefox), Extensions (Ghostery, Privacy Badger, uBlock Origin), Developer Tools.
Practical 9: Data Remanence & Recovery

How-To: Data Remanence & Recovery

  • Purpose: To analyze how deleted data can still be recovered from storage devices.
  • Software Used: Recuva (File recovery), TestDisk (Partition recovery)
  • How to Perform (Part A: Recuva):
    1. Create & delete files from USB drive (Shift+Delete).
    2. Open Recuva ‚Üí Scan ‚Üí recover deleted files.
  • How to Perform (Part B: TestDisk):
    1. Delete a partition using Disk Management.
    2. Open TestDisk ‚Üí select disk ‚Üí Analyze ‚Üí Quick Search.
    3. Recover lost partition and verify.
  • Output: Recovered files and partitions, showing data persistence even after deletion.

üíæ Important Terms & Tools

  • Data Remanence
  • Quick Format vs. Full Format
  • Partition Recovery, File Recovery
  • Cluster / Sector
  • üß∞ Software: Recuva (for files), TestDisk (for partitions), HxD (Hex Editor).
Practical 10: Forensic Report Writing

How-To: Forensic Report for Phishing Incident

  • Purpose: To prepare a complete forensic report for a phishing attack case.
  • Software Used: EnCase Forensic / FTK Imager / Autopsy, MS Word / Google Docs
  • How to Perform:
    1. Simulate phishing attack.
    2. Use EnCase/Autopsy to analyze disk and email artifacts.
    3. Extract evidence: headers, malicious files, logs.
    4. In MS Word, create structured report: Executive Summary, Methodology, Findings, Impact, Recommendations.
    5. Maintain Chain of Custody record for evidence.
  • Output: A complete forensic report documenting the investigation.

  • Indicators of Compromise (IOCs)
